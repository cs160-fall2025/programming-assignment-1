Collaboration policy

Programming assignments should be your own. You are not allowed to copy other students code, or copy/modify large chunks of code or entire projects found online (e.g. on GitHub).

You are allowed to consult online resources, e.g. Stack Overflow, to get answers to specific implementation questions.

LLM-based tools like ChatGPT and GitHub Copilot are helpful and increasingly common in industry, and you may use them to help you complete assignments, with the following caveats:

  • You should stick to using AI for line- or function-level edits; do not paste the entire chunks of the assignment description into ChatGPT (it’s pretty obvious when you do this). Programming assignments are not quite like real-world software development, because we often provide detail in the specification that would not normally be written out for you in a real-world project. If you use these tools, you should practice the skill of communicating with the tools rather than letting the assignment document do this for you.
  • We generally describe what we want you to learn from a particular exercise. Work through each task with the intention of practicing the skills we mention; if AI helped you get unstuck, do your best to come away with an understanding of the concepts targeted by the assignment.
  • When we ask you to think about the design and aesthetics of your application, we’re looking for your creative spark. AI can be great for helping you find the right CSS rules to style an element, but you should be making your own high-level decisions.
  • The goal of these assignments is to develop your skills to work independently on code implementations. LLMs can be part of this process, but when you prompt AI tools to help you write assignment code, you should not share the exact prompts you used with other students. This has a similar effect as sharing code, since it prevents other students from developing their own skills with the tools they are using. Of course, it's always okay to exchange high-level advice with your peers.

Part 3 · Understanding the DOM

Skills: locating elements in the page with the DOM

An HTML document is represented in the browser as a tree data structure. This structure can be accessed and manipulated using JavaScript, through the Document Object Model (DOM) API.

This is a view from my browser's developer tools, showing the DOM structure for a widget from the Grubhub website.

In this task, you'll be asked to write CSS selectors that select various sets of elements from a provided page. Then, you'll write a JavaScript function that (without CSS selectors) uses the DOM tree to select those same elements.

In part3/index.html, you'll find a document from the HTML5 Up template website. Have a look around, and explore the DOM tree using your browser's developer tools! Make sure you're seeing a left sidebar and a main section; you may need to make your browser larger and/or zoom out, since this page has a responsive layout and will change its layout when it becomes too narrow.

Playing around in the Firefox developer tools!

For each of the below tasks, edit code/part3/solution.js, adding a CSS selector that will select all of the required elements on the page. We will use document.querySelectorAll with your CSS selector to verify that the correct elements are selected.

We've added a floating helper panel to the page, which provides buttons you can click to highlight the elements selected by your current solutions (you'll want to reload the page before trying again, when you make a change).

Then, for each task, implement the same selector with pure JavaScript DOM operations. You may not use querySelector or querySelectorAll; instead, use these DOM operations, as needed:

  • getElementById
  • getElementsByClassName
  • getElementsByTagName
  • parentNode
  • childNodes
  • classList

Tip: each of these can be called not just on the global document object, but also on any individual element fetched from the DOM. You'll see an example below, when we call article.getElementsByClassName. This can help you query for elements within a particular scope, rather than searching the entire document.

Note that operations that return multiple elements (like getElementsByClassName) return an HTMLCollection or NodeList, which is different than a typical list (array) in JavaScript. If you're having trouble getting these return values to behave, you can convert them to lists: 

const elements = document.getElementsByClassName("some-class");
const elementsList = [...elements];

// use elementsList like a normal list

We haven't explicitly taught you JavaScript just yet. It's quite a bit like other languages you'll have seen in your technical courses! If you're in need of a bit more guidance, the MDN includes some great detail that should help you pick up JavaScript as a language.

Example: author tags

Each post in the main section (on the right side of the template) contains an "author" section, an <a> tag.

Let's say we want to select all of these author tags associated with main-section posts, except the first one on the page.

For this first example, we'll go through it together (but you should still put this in your submission). You'll complete the subsequent tasks on your own. We've chosen a somewhat complex example to start with so that you can see how to approach the problem!

Let's start by writing a CSS selector. Each of these author tags has the author class, so we could try selecting those: 

.author

This selects too many items, though: it includes author tags not associated with the main-section posts! Let's limit the CSS selector, adding another selector preceding the .author selector.

All posts, including mini-posts, are in an article tag, but only some of these have the post class. These are the ones we want: 

article.post .author

Great! We're still selecting the first author tag, though, and the exercise asked us to avoid that first one. CSS provides a handy pseudo-class for this: 

article.post:not(:first-of-type) .author

That'll do it! That covers the CSS side of things. Now, how do we do this in JavaScript, without using the CSS selector?

Well, let's start by getting all the posts:

const articles = document.getElementsByTagName("article");
for (let i = 0; i < articles.length; i++) {
  const article = articles[i];
  if (article.classList.contains("post")) {
    // This is a post!
  }
}

Now, let's find an author within each post. In this page, the post always contains exactly one author tag, so there's no need to handle the case where no author is found: 

const articles = document.getElementsByTagName("article");
for (let i = 0; i < articles.length; i++) {
  const article = articles[i];
  if (article.classList.contains("post")) {
    const authorTag = article.getElementsByClassName("author")[0]; // always exists
  }
}

And finally, let's keep track of which tags we've found, skipping the first one: 

const selectedElements = [];
let skippedFirst = false;

const articles = document.getElementsByTagName("article");
for (let i = 0; i < articles.length; i++) {
  const article = articles[i];
  if (article.classList.contains("post")) {
    const authorTag = article.getElementsByClassName("author")[0]; // always exists
    if (skippedFirst) {
      // 'push' adds an item to the end of a list
      selectedElements.push(authorTag);
    } else {
      // This is the first one, so we'll skip it
      skippedFirst = true;
    }
  }
}

Now, the selectedElements list contains exactly the elements we selected earlier with CSS.

Put this solution code into code/part3/solution.js. The CSS selector goes in the task0_cssSelector variable (it's just a string), and the JavaScript code goes in task0_jsCode. Make sure to return the final list of selected elements from the function!
